# مدیریت زیرشاخه‌های دروس

این سند توضیح می‌دهد که چگونه می‌خواهیم قابلیت مدیریت زیرشاخه‌های دروس را در سیستم پیاده‌سازی کنیم. هدف این است که کاربر بتواند زیرشاخه‌های یک درس را مدیریت کند (حذف، ویرایش و اضافه کردن).

## قابلیت‌های مورد نیاز

### 1. حذف زیرشاخه از یک درس
کاربر باید بتواند یک زیرشاخه را از درس والد آن جدا کند. در این حالت:
- زیرشاخه حذف نمی‌شود، بلکه `parent_id` آن به `null` تغییر می‌کند
- زیرشاخه به یک درس مستقل تبدیل می‌شود

### 2. اضافه کردن زیرشاخه به یک درس
کاربر باید بتواند از بین دروس موجود، یک یا چند درس را به عنوان زیرشاخه به درس دیگری اضافه کند:
- درس انتخاب شده، `parent_id` آن به شناسه درس والد تغییر می‌کند
- درس به عنوان زیرشاخه نمایش داده می‌شود

### 3. ویرایش زیرشاخه‌ها
کاربر باید بتواند زیرشاخه‌های یک درس را مدیریت کند:
- تغییر ترتیب زیرشاخه‌ها
- انتقال یک زیرشاخه به درس دیگر

## API‌های مورد نیاز

### 1. حذف زیرشاخه
```
PUT /api/lessons/{id}/remove-parent
```

**پارامترها:**
- `id`: شناسه درسی که می‌خواهیم از والد آن جدا شود

**پاسخ موفق:**
```json
{
  "status": true,
  "message": "درس با موفقیت از والد جدا شد",
  "data": {
    "id": 123,
    "title": "عنوان درس",
    "parent_id": null,
    // سایر اطلاعات درس
  }
}
```

### 2. اضافه کردن زیرشاخه
```
PUT /api/lessons/{parent_id}/add-child
```

**پارامترها:**
- `parent_id`: شناسه درس والد
- Body:
```json
{
  "child_id": 456
}
```

**پاسخ موفق:**
```json
{
  "status": true,
  "message": "درس با موفقیت به عنوان زیرشاخه اضافه شد",
  "data": {
    "id": 456,
    "title": "عنوان زیرشاخه",
    "parent_id": 123,
    // سایر اطلاعات درس
  }
}
```

### 3. دریافت لیست دروس قابل اضافه به عنوان زیرشاخه
```
GET /api/lessons/available-for-parent
```

**پارامترها:**
- `exclude_id`: (اختیاری) شناسه درسی که می‌خواهیم از لیست خارج شود (معمولاً درس والد فعلی)

**پاسخ موفق:**
```json
{
  "status": true,
  "data": [
    {
      "id": 789,
      "title": "عنوان درس 1",
      "parent_id": null,
      // سایر اطلاعات درس
    },
    {
      "id": 790,
      "title": "عنوان درس 2",
      "parent_id": null,
      // سایر اطلاعات درس
    }
    // ...
  ]
}
```

## تغییرات رابط کاربری

### 1. نمایش دکمه‌های مدیریت زیرشاخه‌ها
در صفحه نمایش دروس مرتبط، برای هر زیرشاخه باید دکمه‌های زیر اضافه شود:
- دکمه "حذف از زیرشاخه‌ها" - برای جدا کردن زیرشاخه از درس والد
- دکمه "ویرایش" - برای ویرایش اطلاعات زیرشاخه

### 2. اضافه کردن زیرشاخه جدید
در صفحه نمایش دروس مرتبط، یک دکمه "افزودن زیرشاخه" اضافه شود که با کلیک روی آن:
- مودال یا صفحه‌ای باز شود که لیست دروس قابل اضافه به عنوان زیرشاخه را نمایش دهد
- کاربر بتواند یک یا چند درس را انتخاب کند
- با تأیید، دروس انتخاب شده به عنوان زیرشاخه به درس والد اضافه شوند

## نکات پیاده‌سازی

1. **بررسی حلقه در ساختار درختی**: هنگام اضافه کردن زیرشاخه، باید بررسی شود که حلقه در ساختار درختی ایجاد نشود (مثلاً درس A زیرشاخه درس B باشد و درس B زیرشاخه درس A)

2. **به‌روزرسانی نمایش**: پس از هر تغییر (حذف یا اضافه کردن زیرشاخه)، لیست دروس مرتبط باید به‌روزرسانی شود

3. **مدیریت دسترسی‌ها**: بررسی شود که کاربر دسترسی لازم برای مدیریت زیرشاخه‌ها را داشته باشد

4. **تأییدیه قبل از حذف**: قبل از حذف زیرشاخه، از کاربر تأییدیه گرفته شود

## نمونه جریان کاری

1. کاربر روی دکمه "مشاهده دروس مرتبط" برای یک درس کلیک می‌کند
2. لیست دروس مرتبط (زیرشاخه‌ها، والد و خواهر/برادرها) نمایش داده می‌شود
3. کاربر می‌تواند:
   - روی دکمه "حذف از زیرشاخه‌ها" برای یک زیرشاخه کلیک کند
   - روی دکمه "افزودن زیرشاخه" کلیک کند و از لیست دروس موجود، زیرشاخه جدید اضافه کند
   - اطلاعات زیرشاخه را ویرایش کند

با این قابلیت‌ها، کاربر می‌تواند به راحتی ساختار درختی دروس را مدیریت کند.
